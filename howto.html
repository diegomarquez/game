How to GAME-BUILDER
So you decided to give this a try? Good. Let's get started.

Tools

These are the different tools and libraries used in a default Game-Builder project. You should be familiar with these, and install what you don't already have

  1. node
    • Game-Builder relies on bower and grunt to try and make things a little less annoying when developing. Those two are built on node. So you need to install it. This is a pretty nice guide.
  2. git
    • One of the grunt tasks uses git to download Game-Builder, so you need to install it. Here is a guide on how to to that.
  3. requirejs
    • Everything in Game-Builder is a requirejs module. It helps a lot with managing dependencies. This will be downloaded when setting up a new Game-Builder project, so don't worry about it.
  4. grunt
    • Grunt is used to try and solve a few annoying aspects of developing a game for the HTML5 Canvas. This will be downloaded when setting up a new Game-Builder project, so don't worry about it.
  5. bower
    • Bower is used to avoid the need of manually downloading external dependencies. This will be downloaded when setting up a new Game-Builder project, so don't worry about it.

Quick Setup

Quickest way to start doing something

  1. Open your favorite shell
    • We will be using the command line a bit now. Just a little bit.
  2. git clone seed project.
    • https://github.com/diegomarquez/game-builder-seed
  3. npm install in project root.
    • This will install grunt and bower as local dependencies.
  4. grunt in project root.
    • This will download requirejs and Game-Builder source.
  5. Open index.html
    • You should be seeing something in the browser now!

File Overview

A quick overview on the default files of a Game-Builder project.You can view all the files in https://github.com/diegomarquez/game-builder-seed

  • assets/
    • A folder in which to drop assets. It is empty by default. It can be deleted, but it is highly likely that you will be using a folder to put assets into.
  • node_modules/
    • This is created when doing npm install in the root of a project.
  • src/
    • A folder in which to drop source code. It is empty by default. It can be deleted, but it is highly likely that you will be using a folder to put source code into.
  • styles/
    • A folder in which to drop stylesheets. By default it contains main.css. If you change this folder you need to make appropiate changes in index.html.
  • game-builder/
    • This folder is downloaded when doing grunt in the root of a project. It contains all the source that will hopefully help you make a game.
  • lib/
    • This folder is downloaded when doing grunt in the root of a project. By default this is where requirejs is found.
  • tasks/
    • Local grunt that are not good enough to publish them on npm.
  • .bowerrc
    • This file is used to configure bower in different ways. By default it only outlines where should bower drop everything that it downloads.
  • bower.json
    • This file describes all the things that bower should download. By default it will get requirejs and the domready plugin.
  • .gitignore
    • Things that should be ignored by git. By default it ignores the node_modules folder.
  • config.js
    • This file is generatd when doing grunt in the root of a project. It contains configuration for requirejs.
  • Gruntfile.js
    • This contains the grunt tasks available in a Game-Builder project.
  • index.html
    • The main .html file that will be used by a browser.
  • package.json
    • This file is used by npm to install dependencies. It also has other fields used by grunt.

Grunt Tasks

A Game-Builder project has a few grunt tasks available

  • config
    • This task generates requirejs configuration and drops it in config.js.
  • framework
    • This task download Game-Builder source.
  • run
    • This task opens index.html in the default browser.
  • build
    • This task downloads bower dependencies, calls framework and finally calls config.
  • default
    • The default task is just as calling build, but it also calls run
That's pretty much all you need to know about Game-Builder. The rest is making a game. Easier said than done.
