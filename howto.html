How to GAME-BUILDER
So you decided to give this a try? Good. Let's get started.

Tools

These are the different tools and libraries used in a default Game-Builder project. You should be familiar with these, and install what you don't already have

  1. node
    • Game-Builder relies on bower and grunt to try and make things a little less annoying when developing. Those two are built on node. So you need to install it. This is a pretty nice guide.
  2. git
    • One of the grunt tasks uses git to download Game-Builder, so you need to install it. Here is a guide on how to to that.
  3. requirejs
    • Everything in Game-Builder is a requirejs module. It helps a lot with managing dependencies. This will be downloaded when setting up a new Game-Builder project, so don't worry about it.
  4. grunt
    • Grunt is used to try and solve a few annoying aspects of developing a game for the HTML5 Canvas. Install it globally following the instructions on it's website. First you need to install node to do this.
  5. bower
    • Bower is used to avoid the need of manually downloading external dependencies. Install it globally following the instructions on it's website. First you need to install node to do this.
  6. web font loader
    • Web font loader is used to make sure that fonts are completely loaded before doing anything. If you are not using fonts, you don't need to worry about this.

Quick Setup

Quickest way to start doing something

  1. Open your favorite shell
    • We will be using the command line a bit now. Just a little bit.
  2. git clone seed project.
    • https://github.com/diegomarquez/game-builder-seed
  3. npm install in project root.
    • This will install some node dependencies.
  4. grunt in project root.
    • This will download requirejs and Game-Builder source.
  5. Open the index.html file in the project root
    • You should be seeing something in the browser now!

File Overview

A quick overview on the default files of a Game-Builder project.You can view all the files in https://github.com/diegomarquez/game-builder-seed

  • assets/
    • The default folder in which to drop assets. It is empty by default. If you want to use assets from another folder you need to add the proper paths in package.json
  • build/
    • This folder is generated by the build-dev and build-prod tasks and is where the built source for the project is dropped in. The dev folder will have concatenated source and stylesheets, while the prod folder will also minify source and stylesheets
  • config/
    • This folder contains a few files with configuration that has to be set manually if needed
      • font-data.json
        • This .json is the configuration file used by web font loader
      • lib-paths.json
        • This .json is concatenated to the generated generated/config.js file. This should have key value pairs so requirejs knows where to find third party libraries. Common examples would be jQuery or underscore
      • remote-assets.json
        • This .json is concatenated with the generated generated/asset-map.json file. It should have key value pairs to remote resources. This values will be available at runtime through the asset-map module
      • shim-config.json
        • This .json is concatenated to the generated generated/config.js file. It should look like the requirejs shim config
  • generated/
    • This is created when doing grunt in the root of a project. It has two files
      • asset-map.json
        • This .json contains key value pairs with URLs for all the assets that a project will use
      • config.js
        • This is the requirejs shim config configuration
  • node_modules/
    • This is created when doing npm install in the root of a project.
  • src/
    • The default folder where to drop source code. It is empty by default. If you want to use a different folder for your source code, you need to add the proper paths in package.json
  • styles/
    • The default folder in which to drop stylesheets. It has two subfolders. All the contents of this folder are converted into a single stylesheet by the css task
      • css/
        • You can drop regular .css files in this folder
      • less/
        • This folder can have as many subfolders as needed, each of those subfolders must have a file called style.less. Those files will be processed into .css
  • game-builder/
    • This folder is downloaded when doing grunt in the root of a project. It contains all the source that will hopefully help you make a game
  • lib/
    • This folder is downloaded when doing grunt in the root of a project. By default this is where requirejs is found
  • tasks/
    • Local grunt tasks
  • .bowerrc
    • This file is used to configure bower in different ways. By default it only outlines where should bower drop everything that it downloads
  • bower.json
    • This file describes all the things that bower should download
  • Gruntfile.js
    • This contains the grunt tasks available in a Game-Builder project.
  • main.js
    • The main entry point of the application
  • index.html
    • The main .html file that will be used by a browser.
  • package.json
    • This file is used by npm to install dependencies. It also has other fields used by grunt.

Grunt Tasks

A Game-Builder project has a few grunt tasks available. There a more than the ones listed here, but these are the most important.

  • setup
    • This task will get everything a project needs to start running. That includes downloading Game Builder source, downloading bower dependencies, building css and setting up requirejs configuration among other smaller tasks.
  • run
    • This task opens index.html in the default browser.
  • refresh
    • This task is like running setup and then run. It is usefull when adding new files or when work is being done on stylesheets.
  • build-dev
    • This task packages all the source code, stylesheets and and assets into a small folder.
  • build-prod
    • This task is just like build-dev but all the source and stylesheets are minified
  • default
    • The default task is just as calling setup
That's pretty much all you need to know about Game-Builder. The rest is making a game. Easier said than done.
