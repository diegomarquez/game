Errors
Common errors thrown by GAME-BUILDER and explanations on why you are seeing them

Common

Errors that can be thrown by any module

"This method must be overriden"

This one is pretty simple, it means that a method in a parent object was not overriden when extending it.

Reclaimer

"Reclaimer => Missing argument: 'go'"

This error is thrown when calling the claim() method and the first argument is not provided. Make sure to send in a Game Object.

"Reclaimer => Missing argument: 'id'"

This error is thrown when calling the claim() method and the second argument is not provided. Make sure to send in either the typeId or the poolId of the Game Object you are trying to recycle.

Game Object Pool

"Game Object Pool => Game Object with type: 'X' does not have a value for maxAmount. It can not be requested explicitly"

When setting up the Game Object Pool you need to specify the maximun amount of objects of a given type that will be available. Any object type that does not provide that information is assumed to only be able to exist as a child of another type. Child objects can not be requested explicitly. If you need to do that, you need to specify an amount to use in the Game Object Pool.

"Game Object Pool => Game Object with type: 'X' is not available"

The whole idea behind the Game Object Pool is pooling object to avoid excesive creation and destruction of objects. If you see this error it just means the pool run out of objects of the type you requested.

This can mean that you either need more objects of that type in the pool, or that your logic is not behaiving properly.

"Game Object Pool => Configuration with id: 'X' does not exist"

This error should not be seen under normal circumstances. If you are seeing it, it is probably a nasty bug.

Component Pool

"Component Pool => Component with id: 'X' is not available"

The whole idea behind the Component Pool is pooling object to avoid excesive creation and destruction of objects. If you see this error it just means the pool run out of objects of the type you requested.

This can mean that you either need more objects of that type in the pool, or that your logic is not behaiving properly.

Assembler

"Assembler => Game Object with type: 'X' is not a container, can not add childs to it"

If while configuring the Game Object Pool a regular Game Object was set to have children, this error will be thrown when trying to activate it. Too fix it define, it as a Game Object Container.

Path Renderer

"Path Renderer => Missing argument: 'pathWidth'"

pathWidth is a required argument to use a Path Renderer.

"Path Renderer => Missing argument: 'pathHeight'"

pathHeight is a required argument to use a Path Renderer.

Collision Component

"Collision Component => GameObject with typeId: 'X', needs to define an onCollide method, yo."

A Game Object defined to use a Collision Component must define an onCollide(other) method. If you see this error, that is probably what you are missing.

Timer Factory

"Timer Factory => This owner is already using this property, assigning it again with a timer might cause your program to go ape shit."

When creating a Timer you have to specify a property name to which that Timer will be binded in the owner scope. If that property name is already in use in the owner scope, this error will be thrown to alert about incoming undefined behaviour.

"Timer Factory => Timer must have an owner, if unsure just send in 'this'."

owner is a required argument to use a Timer. As the error indicates, if unsure just pass in 'this'.

"Timer Factory => Timer must have an name to identify it later."

name is a required argument to use a Timer. The is used later to apply commands to a specific timer, or to many timers at once.

"Timer Factory => Timer must have a propertyName to be refered with from its owners scope."

propertyName is a required argument to use a Timer. The owner scope gets access to the newly created Timer through this name. The Timer Factory makes use of this property to clear the timer from the owner scope when needed.

Timer

"Timer => Can't modify timer while it is running."

This error is thrown when trying to modify the properties of a Timer while it is running.

State

"State => State with id: 'X' caused an un expected error."

This error is thrown by State objects while attempting a transition. The code is executed in a 'try, catch' block, so any errors in the state code are caught.

Sound Player

"Sound Player => Still loading resources. Wait till everything is complete, before loading more."

This error is thrown by the loadAll() method when attempting to call it before it has finished loading the previous batch of resources.

"Sound Player => Id: 'id' is already in use."

Attempting to use the same id twice will result in this error. Just use different ids for different sound resources.

Font Loader

"Font Loader => No fonts are available."

For whatever reason no fonts are available, if the resources are remote. There is not much to be done about it.

"Font Loader => 'Font: 'X' could not be loaded'."

A specific remote font is not available, not much to do about it.

Layers

"Layer => Layer with id: 'X' does not exist."

Tried to access a Layer which does not exist. Maybe the id is wrong, or maybe the Layer was deleted un intentionally.
