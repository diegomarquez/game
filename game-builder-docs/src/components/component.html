A requireJS module. For use with Game-Builder

Every component extends from the object defined in this module. If you add this to a game-object it will do nothing, so it needs to be extended.

The idea behind components is being able to add logic to a game-object with out hardcoding it in the game-object itself.

If you are crafty enough when writting components you may even be able to share their functionality between completely different game-objects

The Component object extends delegate so it provides a few events to hook into:

START

When it's start method is called

Registered callbacks get the component as argument.

component.on(component.START, function(component) {});


ADD

When it is added to a game-object

Registered callbacks get the component as argument.

component.on(component.ADD, function(component) {});


REMOVE

When it is removed from a game-object.

Registered callbacks get the component as argument.

component.on(component.REMOVE, function(component) {});


RECYCLE

When the parent game-object is sent back to the game-object-pool it triggers this event which sends the component back to the component-pool

Registered callbacks get the component as argument.

component.on(component.RECYCLE, function(component) {});

configure

Configures properties set via the component-pool

This method is important as it applies all the configuration needed for the component to work as expected.

Parameters:

  • args must be an Object.
    (An object with all the properties to write into the component)
configure: function(args) { if (!args) return; for (var ha in args) { if (Util.isObject(args[ha])) { var getter = args[ha]['_get']; var setter = args[ha]['_set']; if (getter || setter) { if (Util.isFunction(getter)) { Util.defineGetter(this, ha, getter); } if (Util.isFunction(setter)) { Util.defineSetter(this, ha, setter); } } else { this[ha] = args[ha]; } } else { this[ha] = args[ha]; } } this.args = args; },

reset

Not so interesting method, it just resets some properties right before the assembler module starts putting together a component.

reset: function() { this.uid = null; this.parent = null; this.enabled = true; },

onStarted

This is called once when the parent game-object is started or when the component is added dynamically to a game-object

Parameters:

  • parent must be an Object.
    (game-object using this component)
onStarted: function(parent) { this.start(parent); this.execute(this.START, this); },

onAdded

This is called by the parent game-object when it adds this component to it's list.

Parameters:

  • parent must be an Object.
    (game-object using this component)
onAdded: function(parent) { this.parent = parent; this.added(parent); this.execute(this.ADD, this); },

onRemoved

This is called by the parent game-object when it removes this component to it's list.

onRemoved: function(parent) { this.removed(parent); this.execute(this.REMOVE, this); this.parent = null; },

onRecycled

This is called by the parent game-object when it is destroying itself.

onRecycled: function(parent) { this.enabled = false; this.recycle(parent); this.execute(this.RECYCLE, this); this.hardCleanUp(); },
