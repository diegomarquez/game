A requireJS module. For use with Game-Builder

Every components extends from the object defined in this module. If you add this to a game-object it will do nothing, so it needs to be extended.

The idea behind components is being able to add logic to the game-object using the components with out hardcoding it in the game-object itself.

If you are crafty enough when writting components you may even be able to share their functionality between completely different game-objects

The Component object extends delegate so it provides a few events to hook into:

added

When it is added to a game-object

Registered callbacks get the component as argument. 

component.on(component.ADDED, function(component) {});

removed

When a component completes.

Registered callbacks get the component as argument.

component.on(component.REMOVED, function(component) {});

recycle

When a component completes.

Registered callbacks get the component as argument.

component.on(component.RECYCLE, function(component) {});

configure Configures properties set via the component-pool

This method is important as it applies all the configuration needed for the component to work as expected.

Parameters:

  • args must be an Object.
    (An object with all the properties to write into the component)
configure: function(args) { if (!args) return; for (var ha in args) { this[ha] = args[ha]; } this.args = args; },

onAdded

This is called by the parent game-object when it adds this component to it's list.

Parameters:

  • parent must be a game-object.
onAdded: function(parent) { this.parent = parent; this.execute(this.ADDED, this); this.added(parent); },

onRemoved

This is called by the parent game-object when it removed this component to it's list.

onRemoved: function() { this.removed(parent); this.execute(this.REMOVED, this); this.parent = null; },

added

Much like onAdded, but this method is only meant to be overriden with out having to remember calling _super()

Parameters:

  • parent must be a game-object.
added: function(parent) {},

removed

Much like onRemoved, but this method is only meant to be overriden with out having to remember calling _super()

Parameters:

  • parent must be a game-object.
removed: function(parent) {},

start

Called by the parent game-object when it is started

Parameters:

  • parent must be a game-object.
start: function(parent) {},

update

Called by the parent game-object after updating itself.

Parameters:

  • delta must be a Number.
    (Time elapsed since last update cycle)
update: function(delta) {},

debug_draw

This method is only executed if the debug property of the parent game-object is set to true. It is better to leave the drawing to the renderer components.

Parameters:

  • context must be a Context 2D.
    (Context 2D property of the Canvas.)
debug_draw: function(context) {},

destroy

Called by the parent game-object when it is sent back to it's pool for reuse.
