A requireJS module. For use with Game-Builder

This is the main thing in Game-Builder. Everything that you do will most likely be an extension of this, or something that creates objects like this one.

Main features are:

Attaching component objects and a renderer object. Why only one renderer? Because I thought it made more sense to have only one renderer for each type of object.

Support for affine transformations thanks to the matrix-3x3 module.

These objects extend delegate so they provide a few events to hook into:

START

When the game object is started

Registered callbacks get the game object as argument 

gameObject.on(gameObject.START, function(gameObject) {});


RECYCLE

When a game object is sent back to the game object pool. This happens before destroying properties in the object.

Registered callbacks get the game object as argument

gameObject.on(gameObject.RECYCLE, function(gameObject) {});


CLEAR

When a game object is sent back to the game object pool. This happens after destroying properties in the object.

gameObject.on(gameObject.CLEAR, function() {});


ADD

When a game object is added to a parent game-object-container The parent is sent to the registered callbacks as argument

gameObject.on(gameObject.ADD, function(parent) {});


REMOVE

When a game object is removed from a parent game-object-container The parent is sent to the registered callbacks as argument

gameObject.on(gameObject.REMOVE, function(parent) {});


ADDTOVIEWPORT

When a game object is added to a viewport + layer pair

An object that looks like this [{viewport: 'ViewportName', layer:'LayerName'}] is sent to all the registered callbacks as an argument

gameObject.on(gameObject.ADD_TO_VIEWPORT, function(v) {});


REMOVEFROMVIEWPORT

When a game object is removed from a viewport + layer pair

An object that looks like this [{viewport: 'ViewportName', layer:'LayerName'}] is sent to all the registered callbacks as an argument

gameObject.on(gameObject.REMOVE_FROM_VIEWPORT, function(v) {});

Game Objects have a bunch of properties

Any of the following properties can be set when configuring the game-object-pool, like so:

gb.goPool.createConfiguration("GameObject_1", "GameObject")

findComponents

Get an object to query the component list of the game object

Returns an Object
(An object to make the query. It has the following methods:
all returns all components that return true for the specified function. Pass no argument to get all components
allWithProp returns all components that have the given property
allWithType returns all components that have the given id in the component-pool
first returns the first component that returns true for the specified function
firstWithProp returns the first component that has the given property
firstWithType returns the first components that have the given id in the component-pool
)
