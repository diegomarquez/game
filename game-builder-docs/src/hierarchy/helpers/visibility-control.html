A requireJS module. For use with Game-Builder

This module provides finer control when deciding which children of a game-object-container should be hidden or shown along with it's parent. The default functionality is all of them, but sometimes you might want something else.

Call the show and hide methods of a game-object-container with a single boolean parameter with a value of true to get a handle of this object.

Ej.

gameObject.hide(true).recurse().not().allWithType('AN_ID'));

The previous example would hide the game-object-cotainer itself along with any children in all of it's heriarchy that did not match the specified type.

VisibilityControl.prototype.toggle = function(uvm) { toggle = true; userVisibilityMethod = uvm; return this; };

recurse

Chain this method to search for children recursively

VisibilityControl.prototype.recurse = function() { recurse = true; return this; };

not

Chain this method to negate the search criteria

VisibilityControl.prototype.not = function() { not = true; return this; };

all

Chain this method to select all the children that return true for the given function

Parameters:

  • f must be a Function.
    (Test function to decide whether a child should change it's visibility. Receives a game-object as argument. Must return true or false.)
VisibilityControl.prototype.all = function(f) { common(all, f); };

allWithProp

Chain this method to search for a matching property name

Parameters:

  • propName must be a String.
    (A property name to search for in the user game-object-container and it's children)
VisibilityControl.prototype.allWithProp = function(propName) { common(withProp, propName); };

allWithType

Chain this method to search for game-objects with a matching poolId or typeId

Parameters:

VisibilityControl.prototype.allWithType = function(type) { common(withType, type); };
