A requireJS module.

This module acts as a hub for the main modules of Game-Builder. So instead of loading them individualy, you just load this one and use the references that it provides.

A bunch of shortcuts

define(['game', 'root', 'groups', 'viewports', 'assembler', 'reclaimer', 'game-object-pool', 'component-pool', 'json-cache', 'error-printer'], function(game, root, groups, viewports, assembler, reclaimer, gameObjectPool, componentPool, jsonCache) { var addToViewPorts = function(go, vports) { var v; if (typeof vports == 'string') { if (this.viewportsAliases[vports]) { v = this.viewportsAliases[vports]; } else { ErrorPrinter.printError('Gb', 'Viewport shortcut ' + vports + ' does not exist.'); } } else { v = vports; } for (var i=0; i<v.length; i++) { viewports.get(v[i].viewport).addGameObject(v[i].layer, go); } }; return { game: game, root: root, groups:groups, viewports: viewports, assembler: assembler, reclaimer: reclaimer, goPool: gameObjectPool, coPool: componentPool, jsonCache: jsonCache, debug: false, viewportsAliases: {},

setViewportShortCut

Store commonly used viewport + layer setups

Parameters:

  • goId must be a String.
    (Id of game-object to add. View game-object-pool, for more details.)

  • groupId must be a String.
    (Id of the group to add the game-object to. View groups, for more details.)

  • vports must be an Array.
    (An array specifying viewports and corresponding layers the game-object should be added to.)

Returns an Object
(The game-object that was just assembled.)

setViewportShortCut: function(alias, vports) { this.viewportsAliases[alias] = vports; },

addToLayer

Wraps all the steps needed to start rendering a game-object

Parameters:

  • goId must be a String.
    (Id of game-object to add. View game-object-pool, for more details.)

  • groupId must be a String.
    (Id of the group to add the game-object to. View groups, for more details.)

  • vports can be an Array or a String.
    (If it is an array specifying viewports and corresponding layers the game-object should be added to. If it is a string, it is used to pick one of the configurations already defined through setViewportShortCut)

Returns an Object
(The game-object that was just assembled.)

add: function (goId, groupId, vports) { var go = assembler.get(goId); groups.get(groupId).add(go); addToViewPorts.call(this, go, vports); go.start(); return go; },

addTextToLayer

This method is basically the same as add but it is used with game-objects that have a text-renderer attached to them.

Parameters:

  • goId must be a String.
    (Id of game-object to add. View game-object-pool, for more details.)

  • groupId must be a String.
    (Id of the group to add the game-object to. View groups, for more details. )

  • text must be a String.
    (String to initialize the text-renderer with.)

  • vports can be an Array or a String.
    (If it is an array specifying viewports and corresponding layers the game-object should be added to. If it is a string, it is used to pick one of the configurations already defined through setViewportShortCut)

Returns an Object
(The game-object that was just assembled.)

addText: function(goId, groupId, text, vports) { var go = assembler.get(goId); groups.get(groupId).add(go); addToViewPorts.call(this, go, vports); go.renderer.text = text; go.start(); return go; },
