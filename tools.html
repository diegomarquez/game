Additional GAME-BUILDER Tools
A little something to try to make development easier. You will need node and git, so go install them!

Generator Game-Builder

https://github.com/diegomarquez/generator-game-builder

A Yeoman generator to scaffold a Game-Builder project. Generators are executed from the command line, so it's convinient to have a shell handy to quickly jump back and forth your text editor

  1. Open your favorite shell
  2. npm install yo -g
    • This will make the yo command globally available
  3. git clone
    • https://github.com/diegomarquez/generator-game-builder
  4. cd generator-game-builder
  5. npm link
    • This will make game-builder a valid argument for yo
  6. mkdir "my_cool_project"
  7. cd "my_cool_project"
  8. yo game-builder
    • This will get all the stuff needed to get started with Game-Builder

Generators Provided

A quick overview on what the main generator and sub-generators do. To create a file where needed with a sub-generator, navigate to the folder you want the file in and run the corresponding command in that folder

  1. yo game-builder
    • The main generator setups a fresh empty project. The first thing you need to do after the scaffolding is complete is run grunt in the project folder
  2. yo game-builder:bundle "bundle name" "module to extend"
    • This sub-generator setups an empty bundle file with name "bundle name". It extends the module with "module to extend". Subsequent arguments are added as dependencies.
    • Ej. yo game-builder:bundle 'my bundle' 'bundle' 'draw' 'util'
  3. yo game-builder:component "component name" "module to extend"
    • This sub-generator setups an empty component file with name "component name". It extends the module with "module to extend". Subsequent arguments are added as dependencies.
    • Ej. yo game-builder:component 'my component' 'component' 'draw' 'util'
  4. yo game-builder:extension "extension name"
    • This sub-generator setups an empty extension file with name "extension name". Subsequent arguments are added as dependencies.
    • Ej. yo game-builder:extension 'my extension' 'draw' 'util'
  5. yo game-builder:game-object "game-object name" "module to extend"
    • This sub-generator setups an empty game-object file with name "game-object name". It extends the module with "module to extend".Subsequent arguments are added as dependencies.
    • Ej. yo game-builder:game-object 'my game object' 'game-object' 'draw' 'util'
  6. yo game-builder:renderer "renderer name" "module to extend"
    • This sub-generator setups an empty renderer file with name "renderer name". It extends the module with "module to extend".Subsequent arguments are added as dependencies.
    • Ej. yo game-builder:renderer 'my renderer' 'renderer' 'draw' 'util'
  7. yo game-builder:state "state name"
    • This sub-generator setups an empty state file with name "state name". Subsequent arguments are added as dependencies.
    • Ej. yo game-builder:state 'my state' 'draw' 'util'
That's all the extra help provided by Game-Builder. Let's hope it is enough.
